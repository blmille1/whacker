# Engine/content separation

Whacker is split into two layers: a content-agnostic game engine (MIT licensed) and game content that plugs into it (CC-BY-4.0 for SRD-derived content, CC-BY-4.0 for original content). The engine handles combat resolution, character state, dice, and event emission — it knows nothing about specific classes, species, monsters, or items. Game content lives in clearly marked directories and provides the data the engine operates on. This separation means the engine can be extracted into its own package or repo independently of any game content, and it keeps SRD-derived content (which carries attribution requirements) mechanically distinct from original code. The trade-off is some indirection — content must conform to engine-defined schemas — but the payoff is a reusable engine and clean licensing boundaries.
